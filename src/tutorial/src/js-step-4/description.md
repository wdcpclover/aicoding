# 类型转换 + 模板字符串 {#js-step-4}

## 1. 自动类型转换的坑

JS 会**自动转类型**，经常出乎意料：

```js
'3' + 2        // '32'  ← 加号有字符串就拼接，不是相加！
3 + '2'        // '32'
'6' - 2        // 4     ← 减号会把字符串转数字
'3' * '4'      // 12
```

经典场景：从输入框拿到的值**永远是字符串**：

```js
const value = input.value   // '20'（字符串）
const age = Number(value)   // 20（数字）
```

## 2. 手动转换

```js
Number('42')         // 42
String(42)           // '42'
parseInt('3.14')     // 3
parseFloat('3.14')   // 3.14
```

## 3. 假值（falsy）

这些在 `if` 里都当 `false`：

```js
if (0)         // false
if ('')        // false
if (null)      // false
if (undefined) // false
if (NaN)       // false
if (false)     // false
```

注意：`'0'`、`[]`、`{}` 都是 truthy（真值）。

## 4. 模板字符串

用反引号 `` ` `` 包裹，可以嵌入 `${...}`：

```js
const name = '小明'
const age = 20
`我叫${name}，今年${age}岁`      // "我叫小明，今年20岁"
`1 + 1 = ${1 + 1}`               // "1 + 1 = 2"
```

ES6 之后，**永远用模板字符串而不是 `+` 拼接**。
