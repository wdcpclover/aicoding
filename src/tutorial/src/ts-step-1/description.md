# TypeScript 是什么 + 为什么用 {#ts-step-1}

> TypeScript = JavaScript + 类型系统。它**不是一门新语言**，而是给 JS 加上「类型保护」，让代码更安全、编辑器更智能、AI 更好用。

## 1. 一句话解释

TypeScript（简称 TS）是 JavaScript 的**超集**——所有 JS 代码都是合法的 TS 代码，但 TS 额外增加了**类型注解**。

```
JavaScript + 类型 = TypeScript
```

## 2. 为什么需要 TS？

JS 是**动态类型**——变量可以随意变类型：

```js
let age = 20      // 是数字
age = '二十岁'    // JS 不报错！
```

大型项目里很容易出 bug：函数期望数字，结果传字符串，**运行时**才发现。

TS 是**静态类型**——**写代码时**就报错：

```ts
let age: number = 20
age = '二十岁'    // ❌ 编辑器立即报错
```

## 3. TS 的好处

| 好处 | 说明 |
|---|---|
| 提前发现 bug | 红色波浪线，不用等运行 |
| 智能提示 | 自动补全属性、方法 |
| 代码即文档 | 看类型就知道函数怎么用 |
| AI 更好用 | 类型让 AI 生成的代码更准确 |
| 大项目必备 | Vue 3、React 都推荐 TS |

## 4. TS 不能直接运行

浏览器只认 JS。所以需要**编译**：

```
TS 代码（.ts）  →  编译器  →  JS 代码（.js）  →  浏览器运行
```

实际开发中，构建工具会自动完成编译，不用手动操作。
