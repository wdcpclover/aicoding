# 结构与语义化 {#html-step-4}

真实网页有成百上千个元素，需要**分块组织**起来。

## 万能容器：div 和 span

```html
<div>一块区域（独占一行，常用来分组）</div>
<span>一小段文字（不换行，常用来标记局部）</span>
```

`<div>` 和 `<span>` 没有特殊含义，纯粹用来"装东西"、方便后面用 CSS 控制样式。

## 语义化标签

与其满屏都是 `<div>`，HTML5 提供了一批**有含义**的标签，让结构一眼就能看懂：

```html
<header>  页头（logo、标题）
<nav>     导航菜单
<main>    页面主体内容
<section> 一个内容区块
<footer>  页脚（版权、联系方式）
```

## 为什么要语义化？

| 好处 | 说明 |
|------|------|
| 易读 | 别人（和 AI）一看标签就懂这块是干嘛的 |
| SEO | 搜索引擎更容易理解页面结构 |
| 无障碍 | 读屏软件能更好地帮助视障用户 |
